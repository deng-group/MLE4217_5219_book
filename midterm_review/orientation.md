# Lecture 1: Orientation and Setup

## What we covered in lecture

- Course arrangement, expectations, and how materials informatics connects data and materials science.
- Practical setup: course directory, VS Code, Python/Jupyter extensions, interpreter selection.
- First-run workflow for Python scripts and Jupyter notebooks.
- Good habits early: backup, password hygiene, and asking for support when blocked.

## What you should be able to do

- Set up a clean working folder for course materials.
- Create and activate a Python environment.
- Select the correct interpreter in VS Code.
- Create a notebook and run cells from top to bottom without kernel errors.
- Explain what this course means by "materials informatics" in your own words.

## Common mistakes we saw in practicals

- Mixing system Python and course environment Python.
- Installing packages in one environment but running notebooks in another.
- Skipping extension setup, then debugging editor problems as if they were code problems.
- Running cells out of order and treating stale variables as correct results.

## Fast self-check

1. Create a new folder and open it in VS Code.
2. Create and activate an environment, then install `numpy`.
3. Open a notebook, run `import numpy as np`, and print `np.__version__`.
4. Restart kernel and rerun all cells successfully.
